
2 ЖҮЙЕНІ ІСКЕ АСЫРУ МЕН ИНТЕРФЕЙСТІ ҚҰРУ

2.1. Қосымшаның архитектурасы мен серверлік бөлігі


«Music App» мобильді қосымшасын бағдарламалық жүзеге асыру барысында кодтың тұрақтылығын, оқылымдылығын және масштабталуын қамтамасыз ету мақсатында «Clean Architecture» (Таза архитектура) қағидаттарына негізделген қабаттық құрылым таңдалды. Flutter фреймворкінде бұл архитектура нақты жауапкершілік аймақтарына бөлінеді: Деректер қабаты (Data Layer), Домендік қабат (Domain Layer) және Презентациялық қабат (Presentation Layer).
Төменде әр қабаттың жобада қалай іске асырылғаны және серверлік логиканың ұйымдастырылуы егжей-тегжейлі сипатталған.
Деректер модельдері және Домендік қабат
Жобаның іргетасы деректер модельдерінен басталады. Dart тілінің объектіге бағытталған (OOP) мүмкіндіктерін қолдана отырып, жүйенің негізгі сущностьтары (entities) сипатталды:
1.	SongModel класы: Жергілікті медиатекадан алынған әндер үшін. Бұл модель SQLite кестесімен тікелей байланысты және `fromMap`/`toMap` әдістерін қамтиды.
2.	SupabaseSong класы: Бұлттық қоймадан келетін әндер үшін арнайы жасалған модель. Оның ерекшелігі – `getPublicUrl` әдісі арқылы алынған тікелей сілтемені қамтуында. Сондай-ақ, файлдың атынан (`Artist - Title.mp3`) метадеректерді автоматты түрде бөліп алатын `fromStorageFile` фабрикалық конструкторы жүзеге асырылған.
3.	UserModel класы: Қолданушының профилін сипаттайтын, Supabase Auth және Database арасындағы байланыстырушы буын.
Күйді басқару (State Management)
Қосымшаның реактивтілігін қамтамасыз ету үшін Provider пакеті қолданылды. Ол «Observer» паттерні негізінде жұмыс істейді: деректер өзгергенде, интерфейс автоматты түрде жаңарады.
•	MusicProvider: Бұл – қосымшаның ең күрделі контроллері. Ол Just Audio плеерінің күйлерін (playing, paused, buffering, loading) бақылайды. Сонымен қатар, ол ойнату кезегін (Queue Management) басқарады: қолданушы «Келесі» батырмасын басқанда, провайдер тізімдегі келесі әнді анықтап, плеерге жібереді.
•	AuthProvider: Авторизация процесін толығымен капсуляциялайды. Интерфейс (UI) тікелей серверге сұраныс жібермейді, ол тек провайдердің `login()` немесе `register()` әдістерін шақырады. Бұл логиканы интерфейстен бөлуге мүмкіндік береді.
Серверлік логика және API интеграциясы
Серверлік сұраныстар арнайы сервистік кластарға (Service Classes) бөлінген. Бұл «Жалғыз жауапкершілік» (Single Responsibility) принципіне сай келеді.
1. SupabaseService (Singleton):
Бұл класс Singleton паттерні арқылы жүзеге асырылған, яғни қосымшаның бүкіл өмірлік циклінде оның тек бір ғана данасы (instance) болады. Негізгі әдістері:
•	`getSongs()`: Storage Bucket-ке сұраныс жіберіп, барлық файлдар тізімін алады (`.list()`). Кейін Dart-тың `where` операторы арқылы тек аудио форматтарды (`.mp3`, `.m4a`) сүзіп алады.
•	`getPublicUrl(path)`: Әр файлдың бірегей қолжетімділік сілтемесін генерациялайды.
2. SupabaseAuthService:
Қолданушы сессияларын басқаруға жауапты. Қателерді өңдеу (Error Handling) осында жүзеге асады: егер сервер "User not found" қатесін қайтарса, сервис оны қолданушыға түсінікті "Қолданушы табылmadı" хабарламасына аударады.
Офлайн режим және кэштеу стратегиясы
Қосымшаның басты артықшылықтарының бірі – интернетсіз жұмыс істеу қабілеті. Бұл SQLite арқылы жүзеге асты. Қосымша іске қосылғанда, ол фондық режимде құрылғыны сканерлейді және барлық табылған музыкалық файлдарды `DeviceSongs` кестесіне индексациялайды. Келесі жолы қолданушы кіргенде, тізім файлдық жүйеден емес, дайын базадан оқылады, бұл жүктеу жылдамдығын 10 есеге арттырады. Сонымен қатар, Альбом мұқабалары `cached_network_image` арқылы кэштеледі, яғни бір рет жүктелген сурет қайтадан трафик жұмсамайды.


2.2. Қолданушы интерфейсінің сипаттамасы және дизайн элементтері


«Music App» мобильді қосымшасының визуалды стилі Google Material Design 3 (Material You) тұжырымдамасына негізделген. Дизайнның басты мақсаты – «Контент – бірінші кезекте» (Content First). Қолданушының назары интерфейс элементтеріне емес, музыкаға, альбом мұқабаларына және орындаушыларға аударылуы тиіс. Қосымша толығымен жүйелік тақырыпқа (System Theme) бейімделген: түнде автоматты түрде қаракөк түстер гаммасына (Dark Mode) ауысады, бұл көзді шаршатпайды және OLED экранды смартфондарда қуатты үнемдейді.
Негізгі экрандардың сипаттамасы:
1. Авторизация экраны (Auth Screen)
Кіру беті минималистік стильде жасалған. Фонда (Background) музыкалық тақырыпқа сай абстрактілі градиент немесе бұлыңғыр сурет қолданылған. Орталықта – логотип және кіру формасы.
Функционалды элементтер:
•	Validation: Email өрісіне нақты пошта форматы енгізілмесе, «Қате формат» ескертуі қызыл түспен шығады.
•	Loading State: Батырманы басқанда, ол айналмалы индикаторға (CircularProgressIndicator) айналады, бұл қолданушыға процестің жүріп жатқанын білдіреді.
2. Кітапхана экраны (Library Screen)
Бұл – қосымшаның негізгі хабы. Экранның жоғарғы жағында «SliverAppBar» орналасқан, ол тізімді төмен айналдырғанда (scroll) кішірейіп, жоғарыға жабысып қалады. `TabBar` арқылы төрт негізгі бөлімге қол жеткізуге болады:
•	Плейлистер: Плиткалы (Grid) көріністегі плейлистер жинағы. Әр плиткада автоматты генерацияланатын мұқаба көрсетіледі.
•	Әртістер: Дөңгелек аватарлармен безендірілген тізім.
•	Альбомдар: Төртбұрышты үлкен карталар.
•	Загруженное (Cloud): Supabase-тен жүктелген әндер. Бұл тізімде әр әннің жанында кішкентай «Бұлт» иконкасы тұрады, бұл оның онлайн екенін білдіреді.
3. Толық экранды плеер (Player Screen)
Интерфейстің ең әдемі және интерактивті бөлігі.
•	Фон: Ағымдағы әннің мұқабасынан түс палитрасын (PaletteGenerator) алып, оны градиент ретінде фонға қояды. Бұл әр әнге бірегей атмосфера береді.
•	Мұқаба (Artwork): Экранның ортасында үлкен, көлеңкелі сурет. Ойнау кезінде ол ақырын үлкейіп, паузада кішірейетін анимация қосылуы мүмкін.
•	Слайдер (Seek Bar): `audio_video_progress_bar` виджеті қолданылған. Ол тректің жүктелген бөлігін (Buffered) ақшыл түспен, ал ойналған бөлігін қанық түспен көрсетеді.
•	Басқару: Плей/Пауза батырмасы ең үлкен өлшемде ортада орналасқан. Қосымша батырмалар: Shuffle (Араластыру), Loop (Қайталау), Favorites (Жүрекше).
4. Профиль және Баптаулар (Profile & Settings)
Бұл модуль қолданушыға қосымшаны толығымен өз талғамына сай бейімдеуге мүмкіндік береді. Скриншоттар негізінде жүзеге асырылған негізгі бөлімдер:

*   **Аккаунт және Қауіпсіздік:** Қолданушының жеке деректері (Email, қолданушы аты) көрсетіледі. Қауіпсіздік қойындысында екі факторлы аутентификацияны (2FA) баптау, белсенді сессияларды басқару және кіру тарихын қарау мүмкіндіктері қарастырылған.
*   **Оформление (Theme & Appearance):** Интерфейстің сыртқы келбетін өзгертуге арналған бөлім. Қолданушы «Түнгі» (Dark), «Жарық» (Light) немесе жүйелік тақырыптарды таңдай алады. Сонымен қатар, қосымшаның акцент түсін (жасыл, көк, күлгін, қызыл, қызғылт сары) және интерфейс тілін (Қазақша, Орысша, Ағылшынша) ауыстыру функциясы бар.
*   **Құпиялылық (Privacy):** Қолданушы өзінің әлеуметтік көрінісін басқарады. «Жария профиль», «Жария плейлистер» және «Тыңдау белсенділігі» (Listening Activity) сияқты қосқыштар (toggles) арқылы басқа қолданушыларға қандай ақпараттың көрінетінін реттейді.
*   **Хабарламалар (Notifications):** Push және Email хабарламаларын басқару орталығы. Жаңа релиздер, ұсыныстар, плейлист жаңартулары және концерттер туралы ескертулерді жеке-жеке қосып-өшіруге болады.

5. Іздеу бөлімі (Search Section)
Іздеу экраны музыканы жанрлар бойынша интуитивті түрде табуға арналған. Экранның жоғарғы жағында мәтіндік іздеу жолағы орналасқан. Төменгі бөлікте музыкалық жанрлар (Pop, Rock, Hip-Hop, Electronic, Jazz, Classical және т.б.) түрлі-түсті, минималистік плиткалар (Tiles) ретінде ұсынылған. Бұл қолданушыға жаңа музыканы көңіл-күйіне қарай жылдам таңдауға көмектеседі.

Навигация жүйесі
Экрандар арасындағы ауысу `GoRouter` пакеті арқылы жүзеге асырылады. Мысалы, егер қолданушы хабарламадағы сілтеме арқылы өтсе, қосымша бірден нақты сол әнді немесе плейлистті ашады. Төменгі навигация панелі (BottomNavigationBar) жоқ болса да, логикалық ауысулар (Transitions) оңнан солға сырғу (Slide) анимациясымен сүйемелденеді, бұл iOS стиліндегі табиғи сезімді тудырады.




ҚОРЫТЫНДЫ


Осы курстық жоба аясында «Music App» музыкалық мобильді қосымшасын әзірлеу бойынша кешенді ғылыми-практикалық жұмыс толығымен жүзеге асырылды. Жобаны орындау барысында қазіргі заманғы мобильді индустрияның талаптары, бағдарламалық қамтамасыз етуді әзірлеудің әдістемелері және бұлттық технологиялардың мүмкіндіктері терең зерттелді. Бұл жұмыс тек техникалық өнімді жасаумен шектелмей, толыққанды ақпараттық жүйенің архитектурасын құру, оны оңтайландыру және нарыққа шығару алдындағы дайындық кезеңдерін қамтыды. Төменде жұмыстың негізгі нәтижелері, қабылданған техникалық шешімдердің тиімділігі және жобаның болашағы туралы кеңейтілген талдау ұсынылады.

1. Жобаның мақсаттары мен міндеттерінің орындалуын талдау

Курстық жобаның негізгі мақсаты – Flutter фреймворкі мен Supabase платформасы негізінде гибридті архитектуралы музыкалық қосымша құру болатын. Бұл мақсатқа жету үшін қойылған барлық міндеттер сәтті шешілді:
*   **Архитектуралық жобалау:** Қосымшаның құрылымы Clean Architecture қағидаттарына сәйкес жасалды. Деректер қабаты (Data Layer), бизнес-логика (Domain Layer) және интерфейс (Presentation Layer) бір-бірінен оқшауланған. Бұл кодтың сапасын арттырып, оны тестілеуді және сүйемелдеуді жеңілдетті.
*   **Гибридті сақтау жүйесі (Hybrid Storage):** Жобаның ең күрделі инженерлік шешімі – офлайн және онлайн режимдердің бірігуі. SQLite жергілікті дерекқоры мен Supabase бұлттық қоймасының синхрондалуы ешкімге байқалмайтын (seamless) түрде жүзеге асты. Қолданушы интернет жоқ кезде құрылғысындағы файлдарды тыңдайды, ал желіге қосылғанда  бұлттық коллекцияға қол жеткізеді.
*   **Медиа-ядроның реализациясы:** Just Audio кітапханасы негізінде жасалған плеер барлық заманауи талаптарға жауап береді: фондық режимде жұмыс істеу (Background Playback), хабарландыру тақтасынан басқару (Notification Controls), аудио фокусты (Audio Focus) өңдеу (мысалы, қоңырау шалғанда музыканың тоқтауы).

2. Технологиялық спектрдің тиімділігін бағалау

Жобада қолданылған технологиялық стек (Tech Stack) өзінің тиімділігін практика жүзінде дәлелдеді. Аналогтармен (React Native, Java/Kotlin) салыстырғанда, Flutter + Supabase комбинациясы келесі артықшылықтарды көрсетті:
*   **Flutter фреймворкі:** Бірыңғай кодтық база (Single Codebase) әзірлеу уақытын шамамен 40%-ға қысқартуға мүмкіндік берді. Dart тілінің қатаң типтеу жүйесі (Static Typing) және Null Safety механизмі қосымшаның жұмыс барысындағы сыни қателерден (Runtime Errors) қорғады. Skia графикалық қозғалтқышының арқасында интерфейс анимациялары 60 FPS жиілігінде тегіс жұмыс істейді, бұл iOS нативті қосымшаларынан еш кем түспейді.
*   **Supabase платформасы:** Firebase-ке қарағанда, Supabase-тің реляциялық (SQL) табиғаты деректерді анағұрлым икемді құрылымдауға мүмкіндік берді. Әсіресе, `users` және `profiles` кестелері арасындағы байланыстарды құру және SQL триггерлерін (Triggers) қолдану арқылы қолданушы тіркелген сәтте профильді автоматты жасау логикасы – серверлік кодты жазбай-ақ күрделі процестерді автоматтандырудың жарқын мысалы.
*   **SQLite дерекқоры:** Мобильді құрылғыда үлкен көлемдегі метадеректерді (мыңдаған өлеңдер тізімі) сақтау үшін NoSQL (Hive, SharedPreferences) шешімдеріне қарағанда, классикалық SQL сұраныстары әлдеқайда жылдам және сенімді болып шықты. Әсіресе, күрделі сұрыптау (ORDER BY artist) және іздеу (LIKE %query%) операцияларында SQLite өзінің артықшылығын көрсетті.

3. Архитектуралық шешімдер және бағдарламалау сапасы

Бағдарламалық кодтың сапасы – жобаның ұзақ мерзімді өміршеңдігінің кепілі. Осы жобада келесі кәсіби тәсілдер қолданылды:
*   **Provider Pattern:** Күйді басқару үшін BLoC немесе Riverpod сияқты күрделі құралдардың орнына, оқуға жеңіл әрі тиімді Provider таңдалды. Бұл `MusicProvider` және `AuthProvider` кластарын интерфейстен толық ажыратуға мүмкіндік берді. Нәтижесінде, интерфейс кодында бизнес-логика мүлдем жоқ, тек деректерді көрсету логикасы қалды.
*   **Dependency Injection (DI):** Сервистер (Services) мен Репозиторийлер (Repositories) жүйеге тәуелсіз түрде енгізілді. Бұл болашақта Supabase-ті басқа серверге (мысалы, AWS немесе өзіндік Python сервері) ауыстырған жағдайда, бүкіл қосымшаны өзгертпей, тек бір ғана сервистік класты ауыстыруға мүмкіндік береді.
*   **Singleton Pattern:** `SupabaseService` және аудио плеер объектілері Singleton ретінде жүзеге асырылды. Бұл жад ресурстарын (RAM) үнемдеуге және қосымшаның әртүрлі экрандарында бірдей деректермен жұмыс істеуге кепілдік береді.

4. Жоба барысында туындаған қиындықтар және оларды шешу жолдары

Курстық жұмысты орындау кедергісіз болған жоқ. Төменде негізгі техникалық проблемалар және олардың шешімдері келтірілген:
*   **RLS (Row Level Security) Саясаты:** Бастапқы кезеңде қолданушылар профиль суреттерін жүктей алмады. Мәселе Supabase-тегі қауіпсіздік ережелерінің дұрыс бапталмауында болды. Шешім ретінде, SQL Editor арқылы жаңа саясат жазылды: «Кез келген аутентификациядан өткен қолданушы файл жүктей алады, бірақ тек өзінің папкасына (`storage/uid/`) өзгерту енгізе алады».
*   **Аудио Латенттілік (Latency):** Интернет жылдамдығы төмен болғанда, бұлттық әндер кідіріспен орындалды. Бұл мәселе `just_audio` кітапханасының буферизациялау параметрлерін (Buffering Strategy) оңтайландыру және `LockCachingAudioSource` класын қолдану арқылы шешілді. Қазір ән ойнап жатқанда, ол параллельді түрде кэштеледі.
*   **Асинхронды операциялардың қатесі:** Бір уақытта бірнеше файлды жүктеу кезінде интерфейс қатып қалу (UI Freeze) жағдайлары кездесті. Бұл барлық ауыр операцияларды `Isolate` (жеке ағын) арқылы немесе `Future.wait` әдісімен оңтайландыру арқылы шешілді.

5. Әлеуметтік-экономикалық маңызы және нарықтық әлеуеті

«Music App» жобасы жай ғана оқу жаттығуы емес, ол нарыққа шығуға дайын MVP (Minimum Viable Product). Қазақстандық контексте бұл жобаның маңызы зор:
*   **Сапалы қазақтілді контент:** Нарықтағы көптеген қосымшалар (Spotify, Apple Music) қазақ тіліне толық аударылмаған немесе жергілікті менталитетке сай келмейді. Бұл қосымша толық локализацияланған және ұлттық музыканы насихаттауға арналған платформа бола алады.
*   **Бәсекеге қабілеттілік:** Офлайн режимнің болуы интернет сапасы тұрақсыз аймақтарда (ауылдық жерлер, жол үсті) қосымшаны Spotify-дан да ыңғайлы етеді.
*   **Экономикалық модель:** Қосымша болашақта «Freemium» моделі бойынша табыс таба алады. Негізгі функциялар тегін, ал жоғары сапалы аудио (Hi-Fi) және жарнамасыз режим ақылы жазылым арқылы ұсынылады.

6. Тұлғалық және кәсіби даму

Бұл курстық жоба студенттің кәсіби қалыптасуындағы маңызды кезең болды. Жұмыс барысында келесі құзыреттіліктер (Hard Skills) игерілді:
1.  **Mobile Development:** Flutter, Dart, Widget Lifecycle, State Management.
2.  **Backend Engineering:** SQL, Supabase, Authentication flow, RLS Security, Cloud Storage.
3.  **UI/UX Design:** Figma-да интерфейс жобалау, пайдаланушы тәжірибесін зерттеу, анимациялар құру.
Сонымен қатар, «Soft Skills» дамытылды: үлкен көлемді ақпаратты талдау, күрделі техникалық мәселелерге шығармашылық шешім табу, уақытты басқару (Time Management) және жобалық құжаттаманы жүргізу.

7. Болашақ даму стратегиясы (Roadmap)

Жоба аяқталғанмен, оның даму әлеуеті зор. Алдағы 6-12 айға арналған техникалық даму жоспары келесідей:
*   **Жасанды интеллект (AI) интеграциясы:**
    Қазіргі таңда музыкалық ұсыныстар – қосымшаның жүрегі. Болашақта Python (FastAPI) негізінде микросервис құру жоспарлануда. Бұл сервис `scikit-learn` кітапханасын қолданып, қолданушының тыңдау тарихын талдайды (Collaborative Filtering) және оған ұнауы мүмкін жаңа әндерді ұсынады.
*   **Cross-Platform Desktop қолдау:**
    Flutter-дің мүмкіндігін пайдаланып, қосымшаны Windows және macOS операциялық жүйелеріне бейімдеу. Бұл үшін интерфейсті үлкен экрандарға (Responsive Layout) ыңғайлау және пернетақтаның қысқа жолдарын (Keyboard Shortcuts) қосу қажет.
*   **Әлеуметтік желі элементтері:**
    WebSockets технологиясы арқылы достардың белсенділігін нақты уақытта көру («Listening Party» функциясы). Қолданушылар бір бөлмеге жиналып, кезекпен музыка қоя алатын синхронды тыңдау режимін жасау.
*   **Төлем жүйелерін қосу:**
    Stripe немесе жергілікті төлем агрегаторларын (Kaspi Pay) интеграциялау арқылы авторлық құқық иелеріне және әртістерге роялти төлеу механизмін қарастыру.


Жоғарыда айтылғандарды тұжырымдай келе, «Music App» жобасы — бұл жай ғана оқу процесінің нәтижесі емес, қазіргі заманғы ақпараттық технологиялар индустриясының ең жоғары стандарттары мен талаптарына толықтай жауап беретін, техникалық тұрғыдан күрделі, ал архитектуралық тұрғыдан сауатты орындалған, бәсекеге қабілетті бағдарламалық өнім. Бұл жұмыстың сәтті жүзеге асырылуы мобильді қосымшаларды әзірлеу саласындағы іргелі білімді ғана емес, сонымен қатар бұлттық есептеулер (Cloud Computing), реляциялық және объектілік деректер қорын басқару (Database Management), сондай-ақ пайдаланушы интерфейсі мен тәжірибесін жобалау (UI/UX Design) сияқты пәнаралық салалардағы терең түсінікті талап етті.

Жобаның әрбір кезеңі — идеяны қалыптастырудан бастап, соңғы тестілеу мен дебаггинге дейін — студенттен жоғары деңгейдегі академиялық дайындықты және инженерлік ойлау жүйесін талап етті. Flutter фреймворкі мен Supabase платформасының сәтті интеграциясы, «Clean Architecture» қағидаттарын қатаң сақтау, сондай-ақ офлайн және онлайн режимдерді біріктіретін гибридті сақтау жүйесінің енгізілуі — осының барлығы студенттің күрделі инженерлік есептерді шешуге қабілеттілігін айқын көрсетеді.

Алынған практикалық және теориялық нәтижелер студенттің (Әбдікәрім Нариман) «Бағдарламалық қамтамасыз ету» мамандығы бойынша білікті, құзыретті маман ретінде толық қалыптасқанын дәлелдейді. Ол тек дайын құралдарды қолданып қана қоймай, жаңа технологиялық шешімдерді іздеуге, оларды сыни тұрғыдан талдауға және өндірістік процестерге бейімдеуге дайын екендігін көрсетті. Бұл курстық жоба — болашақ IT-маманның кәсіби портфолиосындағы салмақты жұмыс болып табылады және оның нақты өндірістік тапсырмаларды орындауға, командада жұмыс істеуге және инновациялық жобаларды басқаруға толық дайындығын айғақтайды.

Сонымен қатар, жобаның әлеуметтік маңыздылығын атап өтпеуге болмайды. Қазақстандық нарықта сапалы, қазақ тіліндегі контентті ұсынатын және жергілікті инфрақұрылымдық ерекшеліктерге (интернет жылдамдығы, құрылғылардың әртүрлілігі) бейімделген отандық өнімдерге сұраныс жоғары. «Music App» қосымшасы осы сұранысты қанағаттандыруға бағытталған маңызды қадам болып табылады.

**Листинг 2. Supabase Storage қызметімен жұмыс класы (supabase_service.dart)**
```dart
class SupabaseService {
  static const String supabaseUrl = 'https://your-project.supabase.co';
  static const String supabaseAnonKey = 'your-anon-key';
  static const String bucketName = 'songs';

  static SupabaseService? _instance;

  SupabaseService._();

  static SupabaseService get instance {
    _instance ??= SupabaseService._();
    return _instance!;
  }

  SupabaseClient get _client => Supabase.instance.client;

  static Future<void> initialize() async {
    await Supabase.initialize(
      url: supabaseUrl,
      anonKey: supabaseAnonKey,
    );
  }

  Future<List<SupabaseSong>> getSongs() async {
    try {
      final List<FileObject> files = await _client.storage
          .from(bucketName)
          .list();

      return files
          .where((file) => file.name.endsWith('.mp3') || file.name.endsWith('.m4a'))
          .map((file) => SupabaseSong.fromStorageFile(file, bucketName))
          .toList();
    } catch (e) {
      print('Error fetching songs: $e');
      return [];
    }
  }
}
```
